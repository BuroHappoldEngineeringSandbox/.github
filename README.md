# .github

This repository contains organisation-wide defaults for GitHub community health files and workflow templates. Files placed here are inherited by any repository in the organisation that does not define its own.

## Contents

| File / Folder | Purpose |
|---|---|
| `CONTRIBUTING.md` | Default contribution guidelines shown to contributors across all repos |
| `CODE_OF_CONDUCT.md` | Organisation-wide code of conduct |
| `PULL_REQUEST_TEMPLATE.md` | Default pull request template |
| `.github/ISSUE_TEMPLATE/` | Issue templates available across all repos |
| `profile/README.md` | Organisation profile shown on the GitHub org page |

### Issue Templates

| Template | Use for |
|---|---|
| `00_bug.md` | Reporting a bug |
| `01_feature_request.md` | Proposing a new feature |
| `02_test_script.md` | Submitting or requesting a test script |
| `03_documentation.md` | Documentation gaps or improvements |
| `04_compliance.md` | Compliance-related issues |
| `05_external_api_changes.md` | Breaking changes from external APIs |
| `06_question.md` | General questions |

## Making Changes

Changes to files in this repository affect all repositories in the organisation that rely on these defaults. Treat changes here with the same care as changes to shared infrastructure — open a pull request and get a review before merging.
